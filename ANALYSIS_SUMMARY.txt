BUG #1: STOP LOSS TRIGGER TOO TIGHT (-$240.29, P0)
─────────────────────────────────────────────────
Impact: 39 trades, 0% win rate
Root Cause: stop_loss_pct threshold at -15% (or similar) triggers on normal volatility
Evidence: 
  • Stop loss exits: 0% WR, all losing
  • Market resolution exits: 93.8% WR, winning
  • Same positions held to resolution would win

Fix:
  File: /opt/polymarket-bot/exit_manager.py
  Change: stop_loss_pct = 0.15 → stop_loss_pct = 0.98
  
  Why: Only exit on catastrophic crash, let positions recover to market resolution

Verification: After fix, no more 'stop_loss' exit reason should appear in new trades

──────────────────────────────────────────────────

Root Cause: SELL signal handler not disabled or re-introduced in recent patch
Evidence:
  • Memory notes: "SELL signals fully disabled" (Bug #13 fix)
  • But bot still executing 151 SELL signal exits
  • Only 40.4% WR (should be 93%+ if held to resolution)

Fix:

BUG #3: HOUR 00:00 CATASTROPHIC (-$187.09, P0)
────────────────────────────────────────────────
Impact: 25 trades during midnight UTC, 0% win rate, -$187.09 (41% of all losses)
Root Cause: Unknown (likely market volatility, signal quality, or data sync issue at midnight)
Evidence:
  • Hour 00:00: 0% WR
  • Hour 01:00: 33% WR
  • Hours 15-20: 100% WR
  • Clear pattern: midnight window is broken

Fix:
  File: /opt/polymarket-bot/run_signal_bot.py, in handle_signal() function
  Change: Add at start:
    hour = datetime.now(timezone.utc).hour
    if hour == 0:
        logger.info("Skipping signals during hour 00:00")
        return

  Why: No good trades at midnight, high variance window, likely system issue

Verification: After fix, zero trades should appear during 00:00-00:59 UTC


BUG #4: ENTRY PRICE MISMATCH (LONG-TERM, P1)
──────────────────────────────────────────────
Impact: 105 trades at 0.80-0.90, 49.5% WR, -$164.52 (sunk cost, but preventable)
Root Cause: Not following asymmetric sizing from DB strategy
Evidence:
  • 0.65-0.80 bucket: 92% WR (high-confidence, big bets)
  • 0.80+ bucket: 51% WR (low-confidence, should be small bets)
  • But sizing is uniform across all entry prices

Fix:
  File: /opt/polymarket-bot/signal_executor.py, in calculate_position_size()
  Change: Add entry price scaling:
    if entry_price > 0.80:
        return min(position_size, 10.0)  # Cap at $10
    elif entry_price < 0.40:
        return min(position_size, 5.0)   # Cap at $5
    else:
        return position_size  # Keep $15-25 for 0.40-0.80

  Why: DB strategy: asymmetric sizing (big bets on high-confidence, small on low-confidence)

Verification: New 0.80+ trades should be smaller; overall performance in that bracket should improve
